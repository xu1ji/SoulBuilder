**关键原则**：
- shared 必须先完成，其他项目依赖它
- assessment、training、animation 可以完全并行开发
- app 最后整合

## 四、开发顺序建议

### Phase 1：接口先行（1小时）
- [ ] shared - 定义所有 TypeScript 接口
- [ ] 所有开发者拉取 shared 作为依赖

### Phase 2：并行开发（3-5天）
- [ ] assessment - 评估引擎
- [ ] training - 训练对话
- [ ] animation - 仪式感动画

### Phase 3：整合（1-2天）
- [ ] app - 整合所有子项目

---

## 五、每个子项目的交付物

### shared
- `types.ts` - 所有接口定义
- `constants.ts` - 常量定义
- `utils.ts` - 共享工具函数

### assessment
- `AssessmentEngine` - 评估引擎类
- `/api/agents/:id/assessment` - 评估 API
- 雷达图组件

### training
- `ConstructSpace` - 纯白训练空间组件
- `TrainingChat` - 对话组件
- SSE 流式接口对接
- 5 种训练模式支持

### animation
- `TrainingAnimation` - 主动画组件
- `CodeRain` - 代码雨背景
- `PixelFighter` - 像素风功夫小人
- 5 阶段动画序列

### app
- 路由配置
- 页面组件
- 状态管理
- 整合所有子项目

## 六、接口契约（核心）

```typescript
// shared/types.ts

// Agent 信息
interface Agent {
  id: string
  name: string
  title: string
  level: number        // 1-5
  score: number        // 0-100
  rank: 'E' | 'D' | 'C' | 'B' | 'A' | 'S'
  lastTrained?: Date
  status: 'new' | 'trained' | 'upgradable'
}

// 评估结果
interface Assessment {
  agentId: string
  totalScore: number    // 0-100
  rank: string          // E-S
  dimensions: {
    identity: number    // IDENTITY.md 完整度
    soul: number        // SOUL.md 完整度
    heartbeat: number   // HEARTBEAT.md
    user: number        // USER.md
    tools: number       // TOOLS.md
    agents: number      // AGENTS.md
    memory: number      // MEMORY.md
  }
  radarData: number[]   // 雷达图数据
}

// 训练结果
interface TrainingResult {
  agentId: string
  oldScore: number
  newScore: number
  skills: string[]
  files: {
    identity: string    // IDENTITY.md 内容
    soul: string        // SOUL.md 内容
    user: string        // USER.md 内容
    tools: string       // TOOLS.md 内容
    agents: string      // AGENTS.md 内容
  }
}

// 动画输入
interface AnimationProps {
  duration: number      // 动画时长（秒）
  skills: string[]      // 注入的技能列表
  beforeScore: number   // 训练前分数
  afterScore: number    // 训练后分数
  onComplete: () => void
}
```

## 七、合并指南

当所有子项目完成后：

1. 将 assessment 的后端代码合并到 app 的 `/server` 目录
2. 将 training、animation 的组件合并到 app 的 `/src/components` 目录
3. 更新路由和状态管理
4. 统一测试
