## 🏗️ 二、 物理路径实操：建立“软链接”群落

为了实现上述架构，请**严格执行**以下命令（以 `researcher` 实例为例）：

### 1. 创建子办公室
```bash
# 1. 建立物理隔间
mkdir -p ~/.openclaw/agents/researcher/memory
```

### 2. 注入“共享基因” (软链接)
```bash
# 2. 软链接共享文档 (确保根目录已有这些文件)
# 这样当你改了根目录的 USER.md，20 个助手会同步“意会”
ln -s ~/.openclaw/USER.md ~/.openclaw/agents/researcher/USER.md
ln -s ~/.openclaw/AGENTS.md ~/.openclaw/agents/researcher/AGENTS.md
```

### 3. 创建“独立灵魂” (独立文件)
```bash
# 3. 为该实例创建独立的人格与心跳清单
touch ~/.openclaw/agents/researcher/SOUL.md
touch ~/.openclaw/agents/researcher/IDENTITY.md
touch ~/.openclaw/agents/researcher/HEARTBEAT.md
touch ~/.openclaw/agents/researcher/MEMORY.md
```

## 🔍 四、 最小可行性验证 (MVT)

启动后，请按以下步骤验证复刻是否成功：

1.  **进程检查**：在宿主机运行 `ls -R ~/.openclaw/agents/researcher`，检查软链接是否正确指向了共享文档。
2.  **路由测试**：分别在两个钉钉群艾特机器人。
3.  **日志穿透**：
    *   看终端输出。如果是 `researcher` 响应，其读取的日志路径应为 `/agents/researcher/SOUL.md`。
    *   修改根目录的 `USER.md`（加一个小绰号），看两个机器人是否**同时**学会了新称呼。如果同时学会，证明“软链接”架构生效。

## 🛡️ 常见问题应对 (Troubleshooting)

*   **Q: 软链接失效报错？**
    *   **A**: 确保 `ln -s` 使用的是绝对路径，或者相对于子文件夹的正确路径。建议使用 `ln -s ~/.openclaw/USER.md ...`。
*   **Q: 修改了 SOUL.md 但没反应？**
    *   **A**: 由于 Node.js 缓存机制，建议执行 `/reload` 命令或重启一键服务。
*   **Q: 找不到 `bindings` 字段？**
    *   **A**: 请确保 `bindings` 是位于 JSON 第一级的数组，且 `agentId` 与 `instances` 中的 Key 完全匹配。
