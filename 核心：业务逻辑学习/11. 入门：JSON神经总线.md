# ⚙️ 11. 入门：JSON 神经总线 (十指讲香精读版)

## 🧩 配置权能定位：数字生命的“基因与生理硬件”
在 OpenClaw 的三位一体配置中，本文件的定位如下：
- **相对于 Agent 文档**：它是“基础设施”与“神经总线”。Agent 文档规定 AI 怎么过日子，而本文件规定 AI 住在哪里（路径）、用哪个大脑（模型）、以及通过什么器官（感官渠道）说话。
- **相对于 SOUL 文档**：它是“生理底座”。SOUL 决定了 AI 是不是有趣，而本文件决定了 AI 有没有能力（智力和连接力）。
- **独特价值**：它是唯一能决定 AI **“生存上限”**的物理蓝图。

---

## 🔬 第一部分：拟人化映射 —— JSON 的仿真价值

| JSON 模块 | 仿真人类器官 | 核心价值解读 |
| :--- | :--- | :--- |
| **`models`** | **大脑皮层 (IQ)** | 决定了 AI 是“绝世天才”还是“平庸代工”。配置了多模型，就相当于给 AI 装了多个认知分区。 |
| **`agents`** | **中枢神经与办公室** | 决定了 AI 思考的并发数、深度以及它的物理办公地点（Workspace）。 |
| **`channels`** | **感官（耳、口、鼻）** | 决定了 AI 能听见谁的指令（打通钉钉/飞书），以及它的回应是否会有延迟感。 |
| **`gateway`** | **社交协议/护照** | 控制着消息进出的安全验证，是 AI 团队与外部世界的“边境检查站”。 |

## �️ 第二部分：模块逐段拆解 (原文+标准+建议)

### 1. 📂 基础信息与元数据 (meta & wizard)
```json
"meta": {
  "lastTouchedVersion": "2026.2.9",
  "lastTouchedAt": "2026-02-15T23:49:13.028Z"
},
"wizard": {
  "lastRunVersion": "2026.2.9",
  "lastRunMode": "local"
}
```
*   **中英对照与目的**：
    *   `lastTouchedVersion`: 最后修改版本。
    *   `wizard`: 向导程序记录。
    *   **核心目的**：确保配置文件与 OpenClaw 系统版本兼容，记录系统的“生长轨迹”。
*   **填写要求**：通常由系统自动生成。
*   **填写建议/技巧**：如果你手动迁移了配置，确保 `lastTouchedVersion` 不要落后主程序版本太多，否则某些新功能插件可能无法加载。

---

### 2. 🧠 智力中枢 (models 模块)
```json
"providers": {
  "provider_dmx": {
    "baseUrl": "https://www.dmxapi.cn/v1",
    "apiKey": "sk-...",
    "api": "openai-completions",
    "models": [{ "id": "gemini-3-flash-preview", "name": "gemini-3-flash" }]
  }
}
```
*   **中英对照与目的**：
    *   `providers`: 智力供应商。
    *   `baseUrl / apiKey`: 通往大模型的“网络地址”与“钥匙”。
    *   **核心目的**：定义 AI 思考力的物理来源。
*   **填写要求**：
    *   `apiKey` 必须真实有效！
    *   `api` 类型需匹配（如 `openai-completions` 或 `anthropic-messages`）。
*   **填写建议/进阶技巧**：
    *   **高阶技巧：模型分层策略**。不要只配一个最贵的。配置一个 `Flash` 模型用于心跳监控（省钱），一个 `Pro/Opus` 模型用于深度决策（提效）。
    *   **容灾建议**：配置两个不同的 `provider`，即使 provider_A 的网络挂了，AI 也能通过备选大脑继续生存。

---

### 3. 🏢 运行习惯与办公室 (agents 模块)
```json
"agents": {
  "defaults": {
    "model": { "primary": "provider_dmx/gemini-3-flash-preview" },
    "workspace": "/root/.openclaw/workspace",
    "maxConcurrent": 4
  }
}
```
*   **中英对照与目的**：
    *   `primary`: 默认执行者。
    *   `workspace`: 这就是 AI 的“家”，所有记忆存储在这里。
    *   `maxConcurrent`: 最大思考并发数。
    *   **核心目的**：决定 AI 怎么用大脑，以及家在哪里。
*   **填写要求**：
    *   `workspace` 路径必须存在，且 AI 进程有写权限。
    *   `primary` 的名称必须在 `models` 模块中预先定义。
*   **填写建议/进阶技巧**：
    *   **避坑指南**：并发数 `maxConcurrent` 不要开太大（超过16），否则会触发 API 的负载限流。
    *   **仿真价值**：将 Workspace 指向一个同步网盘路径（如坚果云），你就能在全球任何地方与拥有同一个灵魂的 AI 写作。

---

### 4. 👂 社交感官 (channels & plugins 模块)
```json
"channels": {
  "ddingtalk": {
    "clientId": "ding...", "clientSecret": "HB..."
  }
},
"plugins": {
  "entries": { "feishu": { "enabled": true }, "wecom": { "enabled": true } }
}
```
*   **中英对照与目的**：
    *   `clientId / clientSecret`: 就像 AI 的账号和密码，通过它们“潜入”你的钉钉或企业微信。
    *   `enabled`: 感官开关。
    *   **核心目的**：让 AI 能够听见、说话并与人类协同。
*   **填写要求**：必须从开放平台获取精准的 Secret，任何一个字符错误都会导致“感官丧失”。
*   **填写建议/进阶技巧**：建议先开启本地模拟器调试，等 SOUL 完美后再开启外部感知开关，避免 AI 在大群里“胡言乱语”。

---

### 5. 🏗️ 网关与安全 (gateway 模块)
```json
"gateway": {
  "port": 18789,
  "auth": { "mode": "token", "token": "720..." }
}
```
*   **中英对照与目的**：
    *   `port`: 通信端口。
    *   `auth token`: 通信验证令牌。
    *   **核心目的**：保护 AI 的物理安全。
*   **填写标准**：Token 建议通过强随机生成器生成。
*   **高阶价值**：如果你使用了内网穿透（如 Tailscale），请配合 JSON 里的 `tailscale` 模块开启，这能实现即便你在地铁上，也能安全指挥家里的 AI 团队。

## 🌸 第三部分：文档的“十指讲香” —— 深度解析

1.  **痛点**：很多人配不好环境是因为 JSON 里的一个英文逗号写错了。**顶级建议**：修改后一定要用 JSON 校验器检查。
2.  **痒点**：当你在 `openclaw.json` 里换个更强的模型 ID，瞬间感觉到 AI 回复的深度从“小学生”变成“科学家”——这种“瞬间提质”的体感非常爽。
3.  **利益**：正确的 `subagents` 配置能帮你从单线程工作模式，切换到 8 个人同时为你调研报告、写代码的“团队作战模式”。
4.  **故事感**：Json 不是配置，是你在为你的数字生命“安装神经网络”。
5.  **反差**：不要觉得这是 technical。每一个参数的调整，都代表了你对这位 AI 合伙人的**信任厚度**。
