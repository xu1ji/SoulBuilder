## 1. 问题背景

  

### 1.1 初始需求

  

用户希望配置多个钉钉机器人，每个机器人绑定不同的 Agent：

- 机器人 A → main agent（通用助手）

- 机器人 B → worker agent（专属助手）

  

### 1.2 遇到的问题

  

**原始插件设计为单账号模式**：

  

```typescript

// 原始 accounts.ts 代码

export function normalizeAccountId(_accountId?: string | null): string {

return DEFAULT_ACCOUNT_ID; // 硬编码返回默认账号

}

  

export function listDingTalkAccountIds(cfg: OpenClawConfig): string[] {

// 只返回一个默认账号

return [DEFAULT_ACCOUNT_ID];

}

```

  

**结果**：无论配置多少个机器人凭证，插件只会识别和使用第一个。

  

### 1.3 错误尝试

  

最初尝试通过 `bindings` 配置用户路由：

  

```json

{

"bindings": [

{ "agentId": "worker", "match": { "channel": "ddingtalk", "peer": { "kind": "direct", "id": "用户ID" } } },

{ "agentId": "main", "match": { "channel": "ddingtalk" } }

]

}

```

  

**失败原因**：这是"用户路由"而非"机器人路由"。用户希望的是不同机器人绑定不同 Agent，而非同一机器人的不同用户路由。

## 2. 架构原理

  

### 2.1 改造前架构

  

```

┌─────────────────┐

│ 钉钉机器人 A │

└────────┬────────┘

│ Stream 连接

▼

┌─────────────────┐

│ OpenClaw │

│ (单账号模式) │

└────────┬────────┘

│

▼

┌─────────────────┐

│ main agent │

└─────────────────┘

```

  

### 2.2 改造后架构

  

```

┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐

│ 钉钉机器人 A │ │ 钉钉机器人 B │ │ 钉钉机器人 C │

│ (accountId: │ │ (accountId: │ │ (accountId: │

│ default) │ │ worker) │ │ assistant) │

└────────┬────────┘ └────────┬────────┘ └────────┬────────┘

│ │ │

│ Stream 连接 1 │ Stream 连接 2 │ Stream 连接 3

▼ ▼ ▼

┌─────────────────────────────────────────────────────────────────┐

│ OpenClaw Gateway │

│ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │

│ │ Account 1 │ │ Account 2 │ │ Account 3 │ │

│ │ (default) │ │ (worker) │ │ (assistant) │ │

│ └──────┬──────┘ └──────┬──────┘ └──────┬──────┘ │

└─────────┼────────────────┼────────────────┼─────────────────────┘

│ │ │

▼ ▼ ▼

┌─────────────┐ ┌─────────────┐ ┌─────────────┐

│ main agent │ │worker agent │ │assistant │

└─────────────┘ └─────────────┘ └─────────────┘

```

  

### 2.3 核心原理

  

1. **多账号支持**：插件配置支持 `accounts` 对象，每个 key 是一个 accountId

2. **独立连接**：Gateway 为每个账号创建独立的 Stream 连接

3. **路由绑定**：通过 `bindings` 配置 accountId 到 agentId 的映射

## 3. 改造步骤

  

### 3.1 文件清单

  

需要修改的文件（位于 `/root/.openclaw/extensions/ddingtalk/src/`）：

  

| 文件 | 修改内容 |

|------|---------|

| `types.ts` | 添加多账号 Schema 定义 |

| `accounts.ts` | 重写账号解析逻辑 |

| `channel.ts` | 更新配置解析和路由逻辑 |

  

### 3.2 修改 types.ts

  

**新增单账号配置 Schema 和多账号支持**：

  

```typescript

import { z } from "zod";

  

// ======================= DingTalk Config Schema =======================

  

/**

* 单个钉钉账户配置

*/

export const DingTalkAccountSchema = z.object({

/** 是否启用 */

enabled: z.boolean().optional(),

/** 账户名称 */

name: z.string().optional(),

/** 钉钉应用 AppKey */

clientId: z.string(),

/** 钉钉应用 AppSecret */

clientSecret: z.string(),

/** 允许的发送者白名单，默认 ["*"] 允许所有人 */

allowFrom: z.array(z.union([z.string(), z.number()])).optional(),

});

  

export type DingTalkAccount = z.infer<typeof DingTalkAccountSchema>;

  

/**

* 钉钉渠道配置 Schema（支持多账户）

*/

export const DingTalkConfigSchema = z.object({

/** 是否启用钉钉渠道 */

enabled: z.boolean().optional(),

/** 账户名称（兼容旧版单账户配置） */

name: z.string().optional(),

/** 钉钉应用 AppKey（兼容旧版单账户配置） */

clientId: z.string().optional(),

/** 钉钉应用 AppSecret（兼容旧版单账户配置） */

clientSecret: z.string().optional(),

/** 允许的发送者白名单（兼容旧版单账户配置） */

allowFrom: z.array(z.union([z.string(), z.number()])).optional(),

/** 多账户配置 */

accounts: z.record(z.string(), DingTalkAccountSchema).optional(),

});

  

export type DingTalkConfig = z.infer<typeof DingTalkConfigSchema>;

```

  

**关键点**：

- 新增 `DingTalkAccountSchema` 定义单个账号结构

- `DingTalkConfigSchema` 保留旧字段以兼容单账号配置

- 新增 `accounts` 字段支持多账号

  

### 3.3 修改 accounts.ts

  

**重写账号解析逻辑**：

  

```typescript

import { DEFAULT_ACCOUNT_ID, type OpenClawConfig } from "openclaw/plugin-sdk";

import type { DingTalkConfig, DingTalkAccount, ResolvedDingTalkAccount } from "./types.js";

import { PLUGIN_ID } from "./constants.js";

  

/**

* 规范化账户 ID（空或 "default" 返回 DEFAULT_ACCOUNT_ID）

*/

export function normalizeAccountId(accountId?: string | null): string {

if (!accountId || accountId === "default" || accountId === DEFAULT_ACCOUNT_ID) {

return DEFAULT_ACCOUNT_ID;

}

return accountId;

}

  

/**

* 列出所有钉钉账户 ID

*/

export function listDingTalkAccountIds(cfg: OpenClawConfig): string[] {

const dingtalkConfig = cfg.channels?.[PLUGIN_ID] as DingTalkConfig | undefined;

if (!dingtalkConfig) {

return [];

}

  

const accountIds: string[] = [];

  

// 检查多账户配置

if (dingtalkConfig.accounts) {

for (const accountId of Object.keys(dingtalkConfig.accounts)) {

const account = dingtalkConfig.accounts[accountId];

if (account.clientId?.trim()) {

accountIds.push(accountId);

}

}

}

  

// 检查旧版单账户配置（兼容）

if (dingtalkConfig.clientId?.trim()) {

if (!accountIds.includes(DEFAULT_ACCOUNT_ID)) {

accountIds.push(DEFAULT_ACCOUNT_ID);

}

}

  

return accountIds;

}

  

/**

* 解析默认钉钉账户 ID

*/

export function resolveDefaultDingTalkAccountId(cfg: OpenClawConfig): string {

const accountIds = listDingTalkAccountIds(cfg);

return accountIds.length > 0 ? accountIds[0] : DEFAULT_ACCOUNT_ID;

}

  

/**

* 解析钉钉账户配置（支持多账户）

*/

export function resolveDingTalkAccount(params: {

cfg: OpenClawConfig;

accountId?: string;

}): ResolvedDingTalkAccount {

const { cfg, accountId } = params;

const dingtalkConfig = cfg.channels?.[PLUGIN_ID] as DingTalkConfig | undefined;

const normalizedAccountId = normalizeAccountId(accountId);

  

// 默认返回值

const defaultResult: ResolvedDingTalkAccount = {

accountId: normalizedAccountId,

enabled: false,

clientId: "",

clientSecret: "",

tokenSource: "none",

allowFrom: ["*"],

};

  

if (!dingtalkConfig) {

return defaultResult;

}

  

// 尝试从多账户配置中获取

if (dingtalkConfig.accounts && dingtalkConfig.accounts[normalizedAccountId]) {

const account = dingtalkConfig.accounts[normalizedAccountId] as DingTalkAccount;

if (account.clientId?.trim() && account.clientSecret?.trim()) {

return {

accountId: normalizedAccountId,

name: account.name,

enabled: account.enabled ?? true,

clientId: account.clientId.trim(),

clientSecret: account.clientSecret.trim(),

tokenSource: "config",

allowFrom: account.allowFrom ?? ["*"],

};

}

}

  

// 兼容旧版单账户配置

if (normalizedAccountId === DEFAULT_ACCOUNT_ID && dingtalkConfig.clientId?.trim()) {

let clientId = "";

let clientSecret = "";

let tokenSource: ResolvedDingTalkAccount["tokenSource"] = "none";

  

if (dingtalkConfig.clientId?.trim()) {

clientId = dingtalkConfig.clientId.trim();

tokenSource = "config";

}

  

if (dingtalkConfig.clientSecret?.trim()) {

clientSecret = dingtalkConfig.clientSecret.trim();

}

  

return {

accountId: DEFAULT_ACCOUNT_ID,

name: dingtalkConfig.name,

enabled: dingtalkConfig.enabled ?? true,

clientId,

clientSecret,

tokenSource,

allowFrom: dingtalkConfig.allowFrom ?? ["*"],

};

}

  

return defaultResult;

}

```

  

**关键点**：

- `listDingTalkAccountIds()` 返回所有配置的账号 ID

- `resolveDingTalkAccount()` 先查多账号配置，再兼容旧版单账号

- 保持向后兼容：旧配置方式仍然有效

  

### 3.4 修改 channel.ts

  

**更新配置解析和路由逻辑**（关键修改点）：

  

```typescript

config: {

...cfg,

...dingtalkConfig,

...dingtalkConfig.accounts,

[normalizedAccountId]: {

...dingtalkConfig.accounts[normalizedAccountId],

},

},

},

// 兼容旧版单账户配置
```

## 4. 配置说明

  

### 4.1 创建多个钉钉机器人

  

在 [钉钉开放平台](https://open-dev.dingtalk.com/) 创建多个企业内部应用：

  

1. 创建应用 → 选择「机器人」能力

2. 记录每个应用的 **AppKey** 和 **AppSecret**

3. 消息接收模式选择 **Stream 模式**

4. 发布应用

  

### 4.2 创建多个 Agent

  

```bash

# 创建 main agent（默认已存在）

# 创建 worker agent

openclaw agents add worker \

--model provider_dmx/glm-5 \

--workspace ~/.openclaw/workspace-work

  

# 创建更多 agent

openclaw agents add assistant \

--model provider_dmx/glm-5 \

--workspace ~/.openclaw/workspace-assistant

```

  

### 4.3 配置 openclaw.json

  

```json

{

"channels": {

"ddingtalk": {

"enabled": true,

"accounts": {

"default": {

"name": "Leo-Bot",

"clientId": "dingxxxxxxxxxxxx1",

"clientSecret": "secret1..."

},

"worker": {

"name": "Worker-Bot",

"clientId": "dingxxxxxxxxxxxx2",

"clientSecret": "secret2..."

},

"assistant": {

"name": "Assistant-Bot",

"clientId": "dingxxxxxxxxxxxx3",

"clientSecret": "secret3..."

}

}

}

},

"bindings": [

{

"agentId": "main",

"match": {

"channel": "ddingtalk",

"accountId": "default"

}

},

{

"agentId": "worker",

"match": {

"channel": "ddingtalk",

"accountId": "worker"

}

},

{

"agentId": "assistant",

"match": {

"channel": "ddingtalk",

"accountId": "assistant"

}

}

],

"agents": {

"list": [

{ "id": "main" },

{

"id": "worker",

"workspace": "/root/.openclaw/workspace-work",

"agentDir": "/root/.openclaw/agents/worker/agent",

"model": "provider_dmx/glm-5"

},

{

"id": "assistant",

"workspace": "/root/.openclaw/workspace-assistant",

"agentDir": "/root/.openclaw/agents/assistant/agent",

"model": "provider_dmx/glm-5"

}

]

}

}

```

  

### 4.4 配置字段说明

  

| 字段 | 说明 |

|------|------|

| `channels.ddingtalk.accounts` | 多账号配置对象，key 为 accountId |

| `accounts.{accountId}.name` | 账号显示名称（可选） |

| `accounts.{accountId}.clientId` | 钉钉应用 AppKey |

| `accounts.{accountId}.clientSecret` | 钉钉应用 AppSecret |

| `bindings[].match.accountId` | 匹配特定账号的消息 |

| `bindings[].agentId` | 路由到的目标 Agent |

  

### 4.5 兼容旧版配置

  

如果只有一个机器人，仍可使用旧版配置：

  

```json

{

"channels": {

"ddingtalk": {

"enabled": true,

"clientId": "dingxxxxxxxxxxxx",

"clientSecret": "secret..."

}

}

}

```

  

插件会自动将其识别为 `default` 账号。

## 6. 常见问题

  

### Q1: 消息仍然路由到 main agent

  

**原因**：使用了错误的机器人发送消息。

  

**解决**：确认使用的是对应 accountId 的机器人。每个机器人有独立的 AppKey，在钉钉中显示为不同的机器人。

  

### Q2: Gateway 启动后机器人没有连接

  

**排查步骤**：

  

```bash

# 检查配置是否正确

openclaw agents list --bindings

  

# 检查日志错误

grep -i "error\|fail" /tmp/openclaw/openclaw-2026-02-19.log | tail -20

  

# 检查凭证是否正确

cat ~/.openclaw/openclaw.json | grep -A 5 "accounts"

```

  

### Q3: 插件修改后不生效

  

**原因**：OpenClaw 缓存了旧代码。

  

**解决**：

  

```bash

# 完全重启 Gateway

kill $(pgrep -f "openclaw-gateway")

sleep 3

nohup openclaw-gateway > /tmp/openclaw-gateway.log 2>&1 &

```

  

### Q4: 如何添加更多机器人

  

**步骤**：

  

1. 在钉钉开放平台创建新机器人，获取 AppKey/AppSecret

2. 在 `openclaw.json` 的 `accounts` 中添加新账号

3. 创建新的 agent：`openclaw agents add newagent --workspace ~/.openclaw/workspace-new`

4. 在 `bindings` 中添加路由规则

5. 重启 Gateway

  

### Q5: 如何查看当前用户的 staffId

  

**方法**：发送消息后查看日志：

  

```bash

tail -f /tmp/openclaw/openclaw-2026-02-19.log | grep "senderStaffId\|收到消息"

```

  

**输出示例**：

  

```

[DingTalk] 收到消息 | 单聊 | Truman(013154533346-1781243805) | 测试消息

```

  

括号内即为 staffId：`013154533346-1781243805`
