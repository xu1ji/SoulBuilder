
本指南旨在提供一套标准化的、具有高鲁棒性的 Agent 备份与协同方案。通过本指南，你可以在几分钟内为任何 OpenClaw 实例配置好“灵魂同步”系统。

---

## 🎯 三大核心目标
1.  **资产抽离**：仅同步不可再生的核心内容，规避 WebDAV 对万级小文件的限速与封锁。
2.  **100% 复活**：一旦系统崩溃，利用备份文件可在新实例上瞬间找回记忆、性格与能力清单。
3.  **深度协同**：打通 Agent 工作区与人类 Obsidian 笔记库，实现 AI 工作可视化与人机异步协作。

## 🛠️ 第一部分：基础设施配置 (rclone)

### 1.1 安装 rclone
在服务器执行官方安装脚本：
```bash
sudo -v ; curl https://rclone.org/install.sh | sudo bash
```

### 1.2 配置坚果云 WebDAV
使用命令行快速创建配置（替换你的账号和应用密码）：
```bash
rclone config create jianguo webdav \
    url https://dav.jianguoyun.com/dav/ \
    vendor other \
    user your_email@example.com \
    pass your_app_password
```
*Tips: 必须使用坚果云后台生成的“应用密码”，而非主密码。*

## 🧠 第二部分：关键同步策略（聪明之处）

### 2.1 区分“肉身”与“灵魂”
*   **不备份 (肉身)**：`/root/.../openclaw` 程序目录及 `node_modules`。这些是公共代码，随时可重装。
*   **必须备份 (灵魂)**：
    *   `workspace/*.md`: 身份 (`IDENTITY.md`)、性格 (`SOUL.md`)、用户偏好 (`USER.md`)。
    *   `workspace/memory/`: 所有的对话历史。
    *   `workspace/ProjectLibrary/`: 你与 Agent 协作的资产库。
    *   `openclaw.json`: **极其重要**。它是 Agent 的 DNA 蓝图，记录了安装过哪些技能。

### 2.2 排除逻辑 (Exclude Rules)
同步时必须排除以下内容以确保速度：
- `node_modules/` (依赖地狱)
- `skills/` (包含大量 JS 代码，可通过清单一键重装)
- `.obsidian/` (Obsidian 的本地缓存，避免多端冲突)

## 🔄 第三部分：双向协同执行方案

### 3.1 初始化双向同步 (First Run)
在第一次建立连接时，使用 `--resync` 参数强制对齐两端：
```bash
rclone bisync /root/.openclaw/workspace jianguo:/你的云端路径 \
    --include "*.md" \
    --include "memory/**" \
    --include "ProjectLibrary/**" \
    --include "openclaw.json" \
    --exclude "skills/**" \
    --exclude ".obsidian/**" \
    --resync --verbose
```

### 3.2 常态化高频双向同步 (Cron Job)
由于我们已经实现了“极简同步”（文件数 < 100），单次扫描只需约 1 秒。因此，可以将同步频率提升至 **1 分钟一次**，实现接近实时的协同体验。

```bash
# 推荐的高频双向同步命令 (每 1 分钟运行一次)
# 移除了在 WebDAV 上不稳定的 --check-access，增加 --force 以确保鲁棒性
rclone bisync /root/.openclaw/workspace jianguo:/你的云端路径 \
    --filter-from /root/.openclaw/rclone_filter.txt \
    --recover --force --quiet
```
*Tips: 在生产环境中，建议将此命令封装在脚本中并由 crontab 调用。*

## 💡 第四部分：协同工作流 (Obsidian + Agent)

1.  **可视化大脑**：将坚果云目录挂载到 PC 端的 Obsidian 仓库。
2.  **异步协作**：
    *   **人给 AI 发指令**：在 Obsidian 里的项目 MD 写入需求，Agent 下次运行时读取该文件即可获取。
    *   **AI 给人交作业**：Agent 将调研报告、小说草稿直接写入 `ProjectLibrary/`，人类在 Obsidian 里直接润色。
3.  **零 GUI 运行**：Obsidian 的完美界面就是 Agent 的管理后台。

---

## ⚠️ 核心注意事项
1.  **文件冲突**：若两端同时修改同一个 MD，`bisync` 会生成 `.conflict` 文件。建议遵循“谁当前负责该项目谁修改”原则。
2.  **封锁预防**：切勿将同步频率设为“秒级”。WebDAV 协议对高频请求非常敏感。15-30 分钟是安全平衡点。
3.  **恢复流程**：
    - 在新机器安装 OpenClaw。
    - 将备份的 `openclaw.json` 放回 `.openclaw/` 目录。
    - 将 `workspace/` 下的所有 MD 拷回。
    - 运行 `openclaw`，它会自动发现缺失技能并提示安装，完成 100% 复原。
