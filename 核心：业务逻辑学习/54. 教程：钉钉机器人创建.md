# 🤖 54. 教程：钉钉机器人创建与 Stream 模式配置全攻略

> [!TIP]
> **Stream 模式 (流式连接)** 是 2026 年 OpenClaw 部署的首选方案。它最大的价值在于 **“五零”优势**：无需公网 IP、无需域名、无需内网穿透、无需防火墙白名单、无需 SSL 证书。

---

## 🛠️ 第一阶段：创建企业内部应用

1.  **登录平台**：访问 [钉钉开放平台 (open-dev.dingtalk.com)](https://open-dev.dingtalk.com/)。
2.  **创建应用**：
    *   点击 **“应用开发”** -> **“企业内部开发”**。
    *   点击 **“创建应用”**。
    *   **应用类型**：选择“H5微应用”或“小程序”（建议选 H5 即可，实质上我们只用它的机器人能力）。
    *   填写应用名称（如：OpenClaw_Group）、应用描述，上传图标。
3.  **记录凭证 (关键)**：
    *   在 **“凭证与基础信息”** 页面，保存 **AppKey**（对应 OpenClaw 的 `clientId`）和 **AppSecret**（对应 `clientSecret`）。

---

## 🤖 第二阶段：开启机器人能力与 Stream 模式

1.  **添加能力**：
    *   在左侧菜单点击 **“应用能力”** -> **“添加应用能力”**。
    *   找到 **“机器人”** 模块，点击“添加”。
2.  **配置机器人**：
    *   **消息接收模式**：在页面底部找到“消息接收模式”，务必选择 **“Stream 模式”**。
    *   **机器人预览图**：建议上传一张带有“大龙虾”特征的头像以便识别。
3.  **开启权限 (核心)**：
    *   点击左侧 **“开发配置”** -> **“权限管理”**。
    *   搜索并开通以下权限（必选）：
        *   `企业内机器人发送消息权限` (基础通信)
        *   `Card.Instance.Write` (互动卡片写权限)
        *   `Card.Streaming.Write` (AI 打字机效果流式写权限)

---

## 🚀 第三阶段：发布与上线

1.  **版本管理**：
    *   点击左侧 **“版本管理与发布”**。
    *   点击 **“创建新版本”**，填写版本描述。
2.  **设置范围**：
    *   设置可使用该机器人的范围（建议选择“全员”或您所在的测试群组成员）。
3.  **正式发布**：
    *   点击 **“保存并发布”**。此时你的 `AppKey` 正式生效，可以用于 OpenClaw 物理连接。

---

## ⚙️ 第四阶段：在 52 文档中的复刻应用

当你完成了上述 **N 个机器人**（如 Researcher-Bot 和 Writer-Bot）的创建后，你将获得 N 组 `AppKey/AppSecret`。

**在 `openclaw.json` 中的填入方式：**

```json
"channels": {
  "ddingtalk": {
    "enabled": true,
    "accounts": [
      {
        "id": "research-bot", // 自定义 ID
        "clientId": "从步骤一获取的 AppKey",
        "clientSecret": "从步骤一获取的 AppSecret",
        "connectionMode": "stream"
      }
    ]
  }
}
```

## 💡 专家提示 (Pro Tips)

*   **免重启调试**：钉钉开发者后台修改权限或机器人头像后，**通常不需要**重启 OpenClaw 进程，但建议重新发布一次钉钉应用版本以加速缓存更新。
*   **群 ID 获取**：机器人发布后，将其拉入群聊。在 OpenClaw 的运行日志中，搜索 `Incoming message`，你可以直接看到 `chatId`，将其填入 `bindings` 即可锁定群聊响应。
*   **连接断开排查**：如果日志显示连接被重置，请检查：
    1.  AppKey/Secret 是否填错；
    2.  是否在后台点击了“Stream 模式”保存；
    3.  Mac 端防火墙是否限制了 Outbound 的 443 端口连接。
