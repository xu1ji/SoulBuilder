# 56. 实战：OpenClaw 多 Agent 钉钉分身术指南 (2.0 终极版)

> **前言**：本文档针对 **钉钉 (DingTalk)** 渠道进行了全流程重构。相比于 Feishu，钉钉的 Stream 模式（WebSocket）配置更简洁且无需公网 IP。本教程包含开篇准备、目录规范、钉钉配置、以及分身路由，确保 100% 可复刻。

---

## 🏗️ 第一阶段：核心准备（环境与目录）

在执行任何 Agent 逻辑之前，先确保基础环境稳固。

### 1. 软件环境
*   **Node.js**: 22.x 或更高版本。
*   **OpenClaw 核心**: 确认已安装 `openclaw-core`。

### 2. 预创建工作空间 (Best Practice)
虽然命令行可以自动创建，但**手动预创建**是防止权限报错的最佳实践：
```bash
# 建议在用户目录下统一管理
mkdir -p ~/.openclaw/workspace-main
mkdir -p ~/.openclaw/workspace-work
```

---

## 🤖 第二阶段：钉钉开放平台配置 (重要)

无需内网穿透或域名，只需在[钉钉开放平台](https://open-dev.dingtalk.com/)完成以下操作：

1.  **创建应用**：创建“企业内部应用”，类型选择“机器人”。
2.  **获取凭证**：记录 **AppKey** (Client ID) 和 **AppSecret** (Client Secret)。
3.  **开启机器人**：在应用能力中开启“机器人”功能。
4.  **接收模式**：**必须选择“Stream 模式”**。这是保证 local 端直接连接钉钉且无需配置 URL 的核心。
5.  **添加权限**：在“权限管理”中搜索并添加 `Card.Streaming.Write` (卡片流式更新)。
6.  **发布**：点击“版本管理与发布”，发布你的机器人应用。

---

## 📦 第三阶段：招募分身（Agent 配置）

### 1. 命令行初始化
```bash
# 创建主大脑 Agent
openclaw agents add main \
    --model deepseek/deepseek-r1 \
    --workspace ~/.openclaw/workspace-main

# 创建执行侧 Agent (分身角色)
openclaw agents add worker \
    --model zai/glm-4-plus \
    --workspace ~/.openclaw/workspace-work
```

### 2. 设置专属身份 (Emoji 区分)
```bash
openclaw agents set-identity --agent main --name "全能指挥官" --emoji "🎖️"
openclaw agents set-identity --agent worker --name "执行小助手" --emoji "🛠️"
```

---

## ⚙️ 第四阶段：`openclaw.json` 整合配置

将钉钉凭证与 Agent 路由绑定。修改 `~/.openclaw/openclaw.json`：

### 1. 钉钉渠道定义
```json
{
  "channels": {
    "dingtalk": {
      "enabled": true,
      "clientId": "你的AppKey",
      "clientSecret": "你的AppSecret",
      "connectionMode": "stream",
      "groups": {
        "oc_ding_group_id_1": {
          "requireMention": false 
        }
      }
    }
  }
}
```

### 2. 路由分身绑定 (Bindings)
将不同的群 ID 绑定给不同的 Agent，实现一人一办公室：
```json
{
  "bindings": [
    {
      "agentId": "main",
      "match": {
        "channel": "dingtalk",
        "peer": { "kind": "group", "id": "指挥官群ID" }
      }
    },
    {
      "agentId": "worker",
      "match": {
        "channel": "dingtalk",
        "peer": { "kind": "group", "id": "搬砖群ID" }
      }
    }
  ]
}
```

## 📡 第五阶段：内线电话机制 (Agent 协作)

如果你希望在“指挥官群”里让主脑指挥“执行小助手”，需开启 `agentToAgent`：

```json
{
  "tools": {
    "agentToAgent": {
      "enabled": true,
      "allow": ["main", "worker"]
    }
  }
}
```
**实操**：在指挥官群输入 `请让 worker 把昨天的周报汇总一下。` 系统会自动唤起 worker 工作区内容并通过 main 反馈给你。

## 🏁 落地闭环检查表 (Checklist)

1. [ ] **Stream 模式确认**：检查钉钉后台是否显示“已上线”状态。
2. [ ] **目录权限确认**：执行 `ls -la ~/.openclaw` 确认各目录已生成。
3. [ ] **ID 获取**：第一次在群里发言，从控制台日志抓取正确的 `oc_...` 群 ID 并填回配置。
4. [ ] **模型可用性**：确保 `~/.openclaw/agents/[name]/agent/models.json` 中的 API Key 已正确配置。

---
**补全细节**：如果 `openclaw agents add` 失败，请检查环境变量 `PATH` 是否包含 `openclaw` 二进制文件，或者改用全路径调用。
