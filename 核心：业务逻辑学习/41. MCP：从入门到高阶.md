# 41. MCP：从入门到高阶全量指南

本指南定义了 OpenClaw 数字员工如何通过 **Model Context Protocol (MCP)** 获取“通用接口能力”。如果说 `11. JSON 神经总线` 是血管和神经，那么 MCP 就是**标准化的设备驱动**，让 AI 可以像插拔 USB 一样瞬间获取外部世界的读写、搜索与存取能力。

---

## 📖 第一部分：MCP 的本质——AI 时代的“通用插槽”

### 1. 为什么需要 MCP？
传统的 AI 插件需要为每一个模型、每一个平台重写一套代码。**MCP (Model Context Protocol)** 由 Anthropic 发起，它将这种混乱终结为一套**开放标准**。它被形象地称为**“AI 界的 USB-C”**：
*   **跨平台通用**：一套 MCP Server，可以在 OpenClaw、Claude Desktop、Cursor 等任何支持 MCP 的客户端上无缝运行，实现“一次编写，到处调遣”。
*   **安全可控**：MCP 运行在独立的进程中，通过 **JSON-RPC 2.0** 通讯，AI 只能调用你授权的指令（如：权限粒度可精确到具体目录），避免敏感数据暴露。
*   **实时性**：突破了 LLM 的静态数据限制，让 AI 拥有了“实时眼睛”和“执行手臂”。

### 2. 三层技术架构
*   **Host (宿主)**：如 OpenClaw Gateway、Cursor、Claude Desktop。它是 MCP 的运行环境，负责管理权限。
*   **Client (客户端)**：位于宿主内部，通过管道 (STDIO) 或网络 (HTTP/SSE) 与 Server 对话。
*   **Server (服务端)**：具体的“能力提供者”。例如，一个 GitHub Server 提供了 `search_repo` 这个工具。

### 3. 全平台支持矩阵 (2025 最新)
根据实测与专家调研，以下工具已支持 MCP 协议：
*   **AI IDE**: Cursor, Windsurf, Trae, Continue, Cline (完全支持); Zed (部分支持)。
*   **桌面客户端**: Claude Desktop, Cherry Studio, 5ire, ChatWise, Witsy, DeepChat (完全支持)。
*   **网页版平台**: Replit, 百度智能云千帆, LibreChat, NextChat (完全支持); 扣子 Coze (仅支持自有 MCP)。
*   **手机客户端**: 百度心响 (安卓, 有限支持), Operit AI (安卓, 有限支持), Claude App (通过 SSE 远程调用)。

## ⚙️ 第二部分：入门实战——神经总线的挂载

在 OpenClaw 中配置 MCP 主要在 `11. 入门：JSON神经总线.md` 中完成。

### 1. 配置格式示例 (STDIO 模式)
这是最常用的模式，直接调用本地的 Node.js 或 Python 脚本：
```json
"mcp_servers": {
  "github": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-github"],
    "env": {
      "GITHUB_PERSONAL_ACCESS_TOKEN": "你的TOKEN"
    }
  }
}
```

### 2. 关键配置项解读
*   **command**: 执行程序的命令（如 `node`, `python`, `npx`）。
*   **args**: 运行参数，通常指向具体的 Server 入口文件。
*   **env**: 环境变量。这是**最容易出错**的地方，复杂的 Token 和 API Key 必须放在这里。

### 3. 如何开发自己的 MCP Server？
如果你所在的行业有私有 API，可以使用 **FastMCP** 框架进行快速开发：
*   **FastMCP (Python)**：目前最流行的国产/国际通用开发框架，允许你用几行装饰器代码将 Python 函数封装为 MCP Tool。
*   **应用商店**：随着协议普及，未来会像 App Store 一样，让工具复用变得极简。

### 4. 如何发现最新的 MCP Server？
不要在网上乱搜，去以下两个权威“集散地”：
1.  **Smithery.ai**：MCP 世界的“应用商店”。
2.  **mcp-get.com**：极其简洁的命令行搜索工具。
