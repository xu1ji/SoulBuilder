# 🏗️ 53. 方案：物理多开 (进程隔离型·群落部署版)

## 🎯 目标：真正的“物理隔离”与“业务群落”

**物理多开**指的是通过 **Docker 容器** 或 **Native 多用户/多端口** 运行多个完全不相干的 OpenClaw 进程。

*   **适用场景**：多租户 (给朋友用)、高风险任务隔离 (暴力爬虫)、需强文件隐私的业务组。
*   **核心优势**：**安全性极高**。一个实例中毒/崩溃，即便执行了 `rm -rf /`，也只能删掉它自己的容器目录，无法接触宿主机数据。

---

## 🔬 基本原理：Swarm (群落) 架构

基于 MacMini M4 的硬件优势，我们采用容器化隔离：

1.  **物理墙 (Docker Wall)**：每个实例有独立的 CPU/内存限额，互不干扰。
2.  **独立工作区 (Workspace)**：每个人都有自己的硬盘分区（挂载目录）。
3.  **流量分流**：通过端口 (18801, 18802...) 进行逻辑路由。

---

## 🛠️ 具体操作步骤

### 1. 架构选型：Docker Compose
在 MacMini 上，官方最推荐的物理多开方式是 Docker。
请在根目录下建立 `docker-compose.yml`：

```yaml
version: '3.8'
services:
  # 实例 01：公司财务审计
  finance-agent:
    image: ghcr.io/openclaw/openclaw:latest
    ports: ["18801:18789"]
    volumes:
      - ./nodes/finance/config:/root/.openclaw
      - ./nodes/finance/workspace:/data/finance_data # 只能看到财务数据
    environment:
      - OPENCLAW_AUTH_TOKEN=fin_secure_token

  # 实例 02：全网公开爬虫 (高风险)
  crawler-agent:
    image: ghcr.io/openclaw/openclaw:latest
    ports: ["18802:18789"]
    deploy:
      resources:
        limits:
          memory: 8G # 爬虫需要吃浏览器内存，给多点
    volumes:
      - ./nodes/crawler/config:/root/.openclaw
      - ./nodes/crawler/workspace:/data/tmp_crawl # 只能看到临时区
```

### 2. 独立物理目录预备
```bash
# 必须为每个人准备独立的“家”
mkdir -p nodes/finance/config nodes/finance/workspace
mkdir -p nodes/crawler/config nodes/crawler/workspace
```

### 3. “大脑”共享：跨越物理墙
物理隔离的实例若想省钱提效，必须共享宿主机的 **Ollama**：
*   在各实例的 `openclaw.json` 中，将模型 baseUrl 设为：`http://host.docker.internal:11434/v1`。

## 🛡️ 安全加固 (Pro Tips)

1.  **宿主机用户隔离 (进阶版)**：
    如果不仅想在 Docker 里隔离，还想在 macOS 层面隔离，请创建不同的 macOS 系统用户 (如 `oc_user_01`)，并在该用户下运行 Docker。
2.  **网络熔断**：
    物理多开允许你为高风险实例配置 **代理 (VPN)**，确保爬虫流量不消耗你的主宽带 IP。
3.  **权限分级 (与 51 文档联动)**：
    各实例的 `SECURITY_LEVEL` 环境变量应分别设为 `high` (财务) 或 `restricted` (公开访客)。

---

## ⚠️ 风险与维护
1.  **内存占用膨胀**：20 个物理实例的基准开销 (Node 运行时) 远高于逻辑多开。若内存跌破 8G，请立即执行 `docker-compose down [service_name]`。
2.  **日志爆炸**：20 个实例的日志量巨大，建议配置 Docker 的 `json-file` log-driver 限制单文件大小。
